# AGENTS.md — руководство для AI-ассистентов

## Краткое описание

**Sclon** — консольное .NET-приложение для склонения русских фраз по шести падежам. Реализовано на чистом C# без внешних зависимостей, с использованием правил русской морфологии и словарей исключений.

## Архитектура

```
src/
├── Program.cs                                    # Точка входа, интерактивный loop
└── Morphology/
    ├── Models.cs                                 # Перечисления и модели данных
    ├── RussianAnalyzer.cs                        # Анализ слова (род, часть речи, одушевлённость)
    └── RussianDeclensionEngine.cs                # Движок склонения слов и фраз
```

**Принцип:** каждое слово фразы анализируется (`RussianAnalyzer.Analyze`), затем склоняется через `RussianDeclensionEngine`. Прилагательные согласуются с главным существительным.

## Ключевые файлы

### `src/Morphology/Models.cs`

Содержит все перечисления и класс `WordInfo` — контейнер с морфологической информацией о слове.

**Важные типы:**
- `RussianCase` — 6 падежей (0..5)
- `GrammaticalGender` — Masculine, Feminine, Neuter
- `Animacy` — Animate, Inanimate
- `PartOfSpeech` — Noun, Adjective, Preposition, Conjunction и др.

**`WordInfo`** — ключевой класс. Поля:
- `Original` — исходное слово
- `PartOfSpeech` — часть речи
- `Gender`, `Number`, `Animacy` — морфологические признаки
- `IsSoft`, `IsHissing` — флаги для выбора окончаний
- `IsDeclinable` — `true` для Noun и Adjective

**`CaseHelper`** — русские названия падежей и вопросы (кто/что, кого/чего и т.д.).

---

### `src/Morphology/RussianAnalyzer.cs`

Статический класс для анализа русских слов. Не имеет состояния.

**`Analyze(string word) -> WordInfo`** — главный метод. Определяет:
1. Часть речи (`DeterminePartOfSpeech`) — по окончанию
2. Род (`DetermineGender`) — по окончанию + словарь `GenderExceptions`
3. Одушевлённость (`DetermineAnimacy`) — по списку + суффиксам
4. Флаги мягкости/шипящести (`DetermineSoftHissing`)

**Ключевые словари:**
- `GenderExceptions` — ~80 слов с нестандартным родом:
  - мужские на -а/-я: папа, дядя, дедушка, мужчина, судья...
  - средние на -мя: время, имя, племя...
  - мужские на -ь: дождь, камень, день, учитель, словарь...
  - женские на -ь: ночь, мышь, жизнь, соль, связь, радость...
- `AnimateNouns` — ~35 слов, гарантированно одушевлённых
- `FleetingVowels` — слова с беглой гласной и их основа: `пёс→пс`, `день→дн`, `огонь→огн`...
- `Prepositions`, `Conjunctions` — списки неизменяемых частей речи

**Важные методы:**
- `Tokenize(string phrase)` — разбиение фразы на слова (lowercase, без пунктуации)
- `GetStem(WordInfo)` — основа слова
- `GetDeclensionStem(WordInfo)` — основа с учётом беглых гласных (для склонения)

**Правила определения рода для слов на -ь:**
- `-ость`, `-есть` → женский (радость, честь)
- `-жь`, `-шь`, `-чь`, `-щь` → женский (ночь, мышь)
- `-ль` → женский (соль, сталь), **кроме** `-тель`, `-арь` → мужской (учитель, словарь)
- `-нь` → женский (ткань, осень)
- `-сь` → женский (связь)

### `src/Morphology/RussianDeclensionEngine.cs`

Статический класс со всей логикой склонения.

**Главные публичные методы:**
- `DeclinePhrase(string phrase, RussianCase targetCase)` — склонение всей фразы
- `DeclineNoun(WordInfo info, RussianCase targetCase)` — склонение существительного
- `DeclineAdjective(string adj, Gender, Number, Case, Animacy)` — склонение прилагательного

**Внутренняя структура склонения существительных:**
1. `DeclineMasculine` — мужской род (твёрдая основа, на -й, на -ь)
2. `DeclineNeuter` — средний род (на -о, -е, -ё, -ие)
3. `DeclineFeminineA` — женский/мужской на -а/-я (2-е склонение)
4. `DeclineFeminineSoft` — женский на -ь (3-е склонение)
5. `DeclineMyaStem` — разносклоняемые на -мя
6. `DeclinePut`, `DeclineDitya` — особые случаи

**Особенности склонения:**
- Для мужского рода с беглой гласной используется `GetDeclensionStem` (пёс→пс→пса)
- Для 3-го склонения (ночь, соль) окончания: -и (род, дат, предл), -ью (твор)
- Для слов на -ия (лекция): -ии (род, дат, предл), -ию (вин), -ией (твор)
- Для слов на -ие (здание): -ия (род), -ию (дат), -ии (предл), -ием (твор)

**Склонение прилагательных:**
- `DeclineAdjectiveForm` → диспетчеризация по роду/числу
- Твёрдое склонение: -ый/-ой → -ого, -ому, -ым, -ом
- Мягкое склонение: -ий → -его, -ему, -им, -ем
- Шипящая основа: как мягкое (-его, -ему, -им, -ем)
- Винительный падеж: неодуш. = именительный, одуш. = родительный

**Склонение фраз (`DeclinePhrase`):**
1. Токенизация фразы
2. Поиск главного существительного (первого в списке)
3. Для каждого слова: предлоги/союзы → без изменений, существительные → `DeclineNoun`, прилагательные → `DeclineAdjective` с согласованием от существительного
4. `ReconstructPhrase` — восстановление оригинального форматирования (заглавные буквы, пунктуация)

## Известные проблемы и краевые случаи

### Множественное число
Не поддерживается. Все слова считаются единственным числом.

### Беглые гласные
Словарь `FleetingVowels` неполон. Многие слова с беглой гласной обрабатываются неправильно (например, «сон», «лоб», «лёд» — не добавлены). Нужно добавлять в словарь по мере обнаружения.

### Род заимствованных слов
Неологизмы и заимствования (кофе, пальто, такси) не имеют специальной обработки. Некоторые могут определяться неточно.

### Склонение прилагательных
- Прилагательное «хороший» имеет основу на шипящую (хорош-), склоняется по мягкому типу. Это корректно, но не все прилагательные на -ший/-щий распознаются правильно.
- Прилагательные на -ой (большой, дорогой, молодой) склоняются так же, как на -ый.

### Сложносоставные фразы
Фразы с несколькими существительными (например, «красивый дом и сад») склоняют все существительные, но согласование прилагательных ведётся от первого существительного.

### Одушевлённость
Список `AnimateNouns` может быть расширен. Профессии на -тель, -ист, -ец, -чик и т.д. автоматически считаются одушевлёнными, но это может давать ложные срабатывания (например, «выключатель» → одушевлённый из-за -тель).

## Как расширять

### Добавление новых слов с беглой гласной
```csharp
// В RussianAnalyzer.cs, словарь FleetingVowels:
["сон"] = "сн",
["лоб"] = "лб",
["лёд"] = "льд",
```

### Добавление исключений по роду
```csharp
// В RussianAnalyzer.cs, словарь GenderExceptions:
["кофе"] = GrammaticalGender.Masculine,
["такси"] = GrammaticalGender.Neuter,
```

### Добавление одушевлённых существительных
```csharp
// В RussianAnalyzer.cs, список AnimateNouns:
"лев", "тигр", "заяц",
```

### Добавление новых правил склонения
Методы `DeclineMasculine`, `DeclineFeminineA` и т.д. могут быть дополнены новыми case-вариантами. Например, для поддержки множественного числа нужно добавить параметр `GrammaticalNumber` и соответствующие окончания.

## Тестирование

Сборка и запуск:
```bash
cd src && dotnet build && dotnet run
```

Интерактивный режим принимает фразы построчно. Для быстрого теста набора фраз:
```bash
cd src && echo -e "красивый дом\nхороший учитель\nвыход" | dotnet run
```
